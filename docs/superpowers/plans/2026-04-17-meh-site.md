# Meh. Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `meh` subdomain site — a brutalist-deadpan satire commerce site selling Emotionally Disappointing Gadgets™ — across the shared `specificindustries` Next.js monorepo.

**Architecture:** Follow the established portfolio pattern. Create `src/sites/meh/{config.ts, index.ts, pages/*, data/*}`, register in `src/sites/registry.ts` + `src/sites/subdomains.ts` + `src/app/sitemap.ts`. Commerce via the shared `CartProvider` (enabled by `features.commerce: true`). Navigation uses the existing `MegaMenu` component with `style: "dropdown"` for the Readings group. Images generated via an OpenAI `gpt-image-1` script following the `scripts/generate-carterandfils-images.ts` pattern.

**Tech Stack:** Next.js 15 (App Router), TypeScript, Tailwind CSS v4, Inter font, shared commerce + UI components, OpenAI image-gen for all site imagery.

**Spec:** `docs/superpowers/specs/2026-04-17-meh-site-design.md`

**Testing note:** This portfolio does not have per-site unit tests. Verification is `npx tsc --noEmit` + `npm run lint` + `npm run dev` smoke-test (visit each route, confirm no runtime errors, visually review layout). Each task ends with these verification steps before committing.

```ts
export function getProductBySlug(slug: string): Product | undefined {
  return products.find((p) => p.slug === slug)
}

export function getProductsByCategory(category: ProductCategory): Product[] {
  return products.filter((p) => p.category === category)
}

export function getRelatedProducts(slug: string, limit: number = 3): Product[] {
  const current = getProductBySlug(slug)
  if (!current) return []
  return products
    .filter((p) => p.category === current.category && p.slug !== slug)
    .slice(0, limit)
}
```
