# Domaine Carter & Fils Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a new `carterandfils` subdomain site — a seventh-generation French-American family winery that is actually selling motor oil and automotive fluids. All copy commits 100% to the winery fiction.

**Architecture:** Follow the existing subdomain-site pattern exactly. A self-contained module under `src/sites/carterandfils/` with `config.ts`, barrel `index.ts`, `data/*.ts` for products/leadership/journal, and `pages/*.tsx` rendered through the catch-all route. Commerce feature enabled (cart via `CartProvider` + `localStorage`). Two dynamic routes: `/cellar/[slug]` and `/journal/[slug]`.

**Tech Stack:** Next.js 15 (App Router), TypeScript, Tailwind CSS v4, existing shared components (`Hero`, `ProductCard`, `Testimonials`, etc.), `next/font/google` for fonts, OpenAI `gpt-image-1` via `scripts/generate-carterandfils-images.ts` for hero + portraits.

**Spec:** `docs/superpowers/specs/2026-04-16-carterandfils-design.md` — consult for voice rules, palette, and terroir specifics.

**Verification approach:** This codebase does not have a test suite. Each task verifies with `npx tsc --noEmit` (type check) and `npm run lint`. For page-rendering tasks, confirm the route loads in `npm run dev` at `localhost:3000/<path>?site=carterandfils` without runtime errors. Commit after each task.
