# Dehydrated Water Co. — Layout Redesign

## Overview

A structural redesign of the dehydrated water site to visually differentiate it from pigmilk. The current implementation reuses pigmilk's layout patterns (Hero → Grid → Testimonials → CTA). This redesign introduces 10 new shared components and restructures 7 pages so that no two sections on the homepage look alike, and the overall feel is inspired by premium water brands (Open Water, Pure Life, Premium Waters).

The theme, product data, commerce system, and content stay the same. This is a layout and component change, not a content or feature change.

## Goals

- Make the dehydrated water site structurally unique from pigmilk
- Build 10 new reusable shared components for the platform
- Every page should have a distinct visual rhythm — no repetitive Hero → Grid patterns
- Maintain full mobile responsiveness
- Rewrite Privacy and Terms pages with escalating chemistry jargon humor
- All images are 1536x1024 (3:2 landscape, generated by OpenAI) — components must handle this ratio

## Existing Component Modifications

### Hero Component Enhancement

The existing `Hero` component (`src/components/ui/hero.tsx`) needs two additions:

1. **`dark` prop** — When true, renders with `bg-primary` gradient background and white text instead of the default `bg-secondary/30`. All 7 redesigned pages use dark heroes.
2. **`secondaryCtaText` / `secondaryCtaHref` props** — Optional secondary CTA button (outlined/ghost style). The homepage uses dual CTAs: "Shop the Collection" (primary) and "Our Story" (secondary/outlined).

These are backward-compatible additions — existing pigmilk pages are unaffected.

## New Shared Components

All created in `src/components/ui/`. All theme-aware via CSS variables. All responsive.

### 1. WaveDivider

SVG wave separator between sections.

```typescript
interface WaveDividerProps {
  variant?: "wave1" | "wave2" | "wave3"  // Different wave shapes to avoid repetition
  flip?: boolean                          // Vertical flip
  fromColor?: string                      // Top color (defaults to primary)
  toColor?: string                        // Bottom color (defaults to background)
}
```

- Pure CSS/SVG, no JS needed. Server component.
- 3 distinct SVG path variants for visual variety.
- `flip` mirrors vertically for bottom-of-section use.
- Colors default to theme tokens but can be overridden.

### 2. AnimatedCounter

Large number that counts up when scrolled into view.

```typescript
interface AnimatedCounterProps {
  end: number           // Target number
  label: string         // Text below the number
  prefix?: string       // e.g., "$"
  suffix?: string       // e.g., "+"
  duration?: number     // Animation duration in ms, default 2000
}
```

- `"use client"` component. Uses Intersection Observer to trigger count.
- Number displayed in monospace font for ticking effect.
- Counts from 0 to `end` with easing.
- Label rendered below in body font.

### 3. ComparisonTable

Multi-column comparison grid.

```typescript
interface ComparisonTableProps {
  title?: string
  columns: Array<{ name: string; highlighted?: boolean }>
  rows: Array<{ label: string; values: string[] }>
  footnote?: string
}
```

- Highlighted column gets accent border and bold header.
- Dark background (primary) with light text for high contrast.
- Mobile: horizontally scrollable with sticky first column (row labels).

### 4. ProductCarousel

Horizontal scrollable product strip with navigation arrows.

```typescript
interface ProductCarouselProps {
  title?: string
  children: React.ReactNode  // Typically ProductCard components
}
```

- `"use client"` component. Uses `useRef` for scroll container.
- Left/right arrow buttons visible on desktop, hidden on mobile (swipe).
- Smooth scroll behavior.
- No snap points — free scroll.

### 5. SplitSection

Full-bleed 50/50 image+content block.

```typescript
interface SplitSectionProps {
  image: string
  imagePosition?: "left" | "right"  // Default "left"
  dark?: boolean                     // Dark background with light text
  children: React.ReactNode          // Content for the text half
}
```

- Full viewport width, no max-width container.
- Image uses `object-cover` to handle 3:2 source images in any container shape.
- `dark` mode: text side gets `bg-primary` with `text-white`. Light mode: `bg-background`.
- Mobile: stacks vertically, image above content.

### 6. PromoBanner

Full-width gradient banner with headline and optional CTA.

```typescript
interface PromoBannerProps {
  headline: string
  subtext?: string       // Fine print / disclaimer below headline
  ctaText?: string
  ctaHref?: string
}
```

- Background: linear gradient from accent to primary.
- White text, centered.
- CTA renders as a bordered white button.
- Server component (no interactivity).

### 7. StatStrip

Horizontal row of icon + value + label badges.

```typescript
interface StatStripProps {
  stats: Array<{
    icon: string     // Emoji or text character
    value: string    // Large display value
    label: string    // Small description
  }>
}
```

- Renders as a flex row, centered, with dividers between stats.
- Desktop: horizontal row. Mobile: stacks to single column.
- Values rendered in heading font at large size.
- Server component.

### 8. FeaturedProductSpotlight

Split hero highlighting a single product.

```typescript
interface FeaturedProductSpotlightProps {
  image: string
  eyebrow?: string           // Small label above title (e.g., "New for 2026")
  title: string
  description: string
  ctaText: string
  ctaHref: string
  imagePosition?: "left" | "right"  // Default "right"
}
```

- Similar to SplitSection but purpose-built for product marketing.
- Eyebrow text in accent color, uppercase, tracked.
- Larger title than SplitSection (display size).
- CTA as a primary-colored button.
- Image side has a subtle background tint (secondary).
- Mobile: stacks, image above.

### 9. ProcessFlow

Numbered process steps with varying layouts.

```typescript
interface ProcessStep {
  number: string       // "01", "02", etc.
  title: string
  description: string
  image: string
}

interface ProcessFlowProps {
  steps: ProcessStep[]
}
```

- `"use client"` component (for Intersection Observer scroll triggers, optional).
- Each step renders in a different layout variant, cycling through:
  - **Variant A**: 3-column — large number left, text center, image right
  - **Variant B**: Full-width dark banner, centered text, image as background with overlay
  - **Variant C**: 3-column reversed — image left, text center, large number right
  - **Variant D**: 60/40 split — image taking 60% left, text stacked 40% right
- Connecting vertical line or arrow between steps.
- **Mobile**: All variants collapse to the same layout: large step number → image → title → description, with vertical connecting line between steps.

### 10. CascadeTimeline

Horizontal cascading timeline with progressive indentation.

```typescript
interface CascadeTimelineProps {
  items: Array<{ year: string; description: string }>
}
```

- Desktop: Items flow with progressive right-indentation, each row shifted further right than the previous. Connecting arrows between items create a staircase/waterfall visual.
- Year displayed as a bold badge, description as inline text.
- **Mobile (below `md:`)**: Falls back to the existing vertical Timeline layout — stacked items with year badges and dot indicators. Same visual pattern as the existing `Timeline` component.
- `"use client"` component (for responsive breakpoint detection if needed, or pure CSS with media queries).

## Privacy & Terms Content Rewrite

Both pages are rewritten with an escalating chemistry jargon approach. The legal text starts readable and progressively drowns in water science terminology until it becomes completely impenetrable.

### Tone Escalation Pattern

- **Sections 1-2**: Mostly readable. Normal-ish legal language with water puns. Clearly states this is a satire site and disavows all liability.
- **Sections 3-4**: Water science terms creep in. "Data processing" becomes "reverse-osmotic data parsing." "Your information" becomes "your personally identifiable solutes."
- **Sections 5-6**: Heavy jargon. References to molarity, pH levels, hydrogen bonding, and Van der Waals forces applied to data handling concepts.
- **Sections 7+**: Nearly unreadable. Full chemistry-paper prose. Sentences like "The thermodynamic equilibrium of your consent exists in a superposition of aqueous and anhydrous states until observed by our compliance team, at which point the wavefunction collapses into whatever interpretation benefits us."

### Content Requirements

**Both pages must include:**
- Clear satire disclaimer at the top (readable) and bottom (in impenetrable jargon)
- Statement that data collection is total and comprehensive
- No opt-out mechanism exists ("Your data, once collected, achieves a state of irreversible dissolution within our systems")
- Shipping returns policy: liquid water must be returned in a paper envelope
- Dispute resolution via Ezekiel IV in his study

**Privacy-specific:**
- Aggressive data collection: "We collect everything. Your browsing data undergoes complete molecular disassembly."
- No data deletion: "Data, once dehydrated into our ledger system, cannot be reconstituted into its original personally-identifiable liquid state."
- Cookie policy using water metaphors

**Terms-specific:**
- Acceptance is irrevocable ("like dehydration itself")
- Product disclaimers using chemistry terminology
- Limitation of liability referencing states of matter
- Governing law: "The bylaws of the Drywell Estate as interpreted through the lens of theoretical hydrology"

## Image Handling

All images are 1536x1024 (3:2 landscape) generated by OpenAI. Components handle this via:

- **Product images in square containers**: `object-cover` with `aspect-square` container
- **SplitSection**: `object-cover` with `fill` — 3:2 fits naturally in the 50% width panel
- **ProcessFlow**: `object-cover` in various aspect ratio containers per variant
- **FeaturedProductSpotlight**: `object-cover` in the image panel
- **Team photos**: `object-cover` cropped to circles/squares by TeamMember component
- **Hero/background**: `object-cover` with `fill`, 3:2 works well for wide containers

See `docs/dehydratedwater-image-generation-list.md` for the complete image list with prompts.
